API Documentation

This is the documentation for the API, listing all available endpoints.

Users Endpoints

  • POST /api/users: Add a new user
  • GET /api/users: Retrieve all users
  • GET /api/users/:id: Retrieve a user by ID
  • PUT /api/users/:id: Update a user by ID
  • DELETE /api/users/:id: Delete a user by ID
  • POST /api/users/login: User login to get JWT

News Endpoints

  • POST /api/news: Add a news item (requires JWT)
  • GET /api/news: Retrieve all news items
  • GET /api/news/:id: Retrieve a news item by ID
  • PUT /api/news/:id: Update a news item (requires JWT)
  • DELETE /api/news/:id: Delete a news item (requires JWT)
  • GET /api/news/paginated: Retrieve news with pagination
  • GET /api/news/search: Search for news by title, author, or category
  • GET /api/news/sorted-by-category: Retrieve news sorted by category

Protected Routes

For protected routes, include the following header with a valid JWT:

Authorization: Bearer
